# warket — native iOS

A native **SwiftUI** port of the warket web app: a privacy-first asset
watchlist. It talks to the **same Supabase project** as the web app, so existing
vaults open unchanged. Identity is a 12-word seed phrase, SHA-256–hashed on
device into a `vault_hash` — no accounts, no emails.

> Source web app: repository root (`../`). Full plan & milestones live at
> `Obsidian/Projects/warket-native/mvp-implementation-plan.md`.

## Status

**MVP complete (M0–M5).** Core flow: unlock → lists → assets → full CRUD →
drag-reorder → search/tag filtering, on the brand's embedded fonts. Builds
clean; 6/6 tests pass.

## Features

- **Unlock** — enter or generate a 12-word phrase. The phrase is hashed locally
  and **never leaves the device**; only the hash is sent to Supabase. "Keep me
  signed in" remembers the hash across launches.
- **Lists** — native List *or* 2-column grid (persisted toggle), create / edit /
  delete, search (by list name *or* a contained asset's name/ticker), tag-chip
  filtering, drag-reorder.
- **Assets** — add / edit / delete, search, tag filtering, drag-reorder.
- **Asset detail** — image, summary, **markdown notes** (custom block renderer:
  headings, lists, quotes, code, inline styles), tags, tappable resource links
  with favicons.
- **Asset editor** — name/ticker, summary (250-char limit), markdown notes with
  a live Write/Preview toggle, tags, image URL, resources (add/remove/reorder).
- Haptics, toast banners, dark theme.

## Tech

- SwiftUI · iOS 17+ · Swift.
- [supabase-swift](https://github.com/supabase/supabase-swift) (SPM) — the global
  `x-vault-hash` header drives Supabase Row-Level Security.
- Project generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from
  `project.yml`; the `.xcodeproj` is gitignored.

## First-time setup

1. **Install XcodeGen** (one-time): `brew install xcodegen`
2. **Add Supabase credentials** — copy the template and paste the real values
   from the web app's `.env` (`VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY`):
   ```bash
   cd warket-ios
   cp warket/Config/Secrets.xcconfig.example warket/Config/Secrets.xcconfig
   # then edit warket/Config/Secrets.xcconfig
   ```
   `Secrets.xcconfig` is gitignored.
3. **Generate the Xcode project**: `xcodegen generate`
4. **Open & run**: `open warket.xcodeproj`, pick an iPhone simulator, press ⌘R.

After adding/removing files or changing project config, re-run `xcodegen generate`.

## Hash parity (critical)

The Swift hash must match the web app's JS hash **byte-for-byte**, or existing
vaults won't open. `warketTests/HashParityTests.swift` asserts the reference
vectors — it must pass.
